# 室友（诸葛洁涛）工作成果 —— CS2 饰品数据可视化与基础架构

## 概述

负责项目的数据采集、图表可视化、磨损系数计算以及独立的神经网络价格预测模型。从 API 接口封装到前端图表展示，打通了数据到可视化的全链路。

---

## 核心成果

### 1. K 线数据接口封装（`csqaq_api.py`）

封装了 CSQAQ 官方 API 的核心数据接口，为所有可视化功能提供数据基础：

| 接口函数 | 说明 |
|:---|:---|
| `get_chart_data()` | K 线图数据（时间戳、价格、最高价、最低价、成交量） |
| `get_skin_statistic()` | 饰品存世量统计数据 |
| `get_market_index()` | 大盘指数 K 线数据（近 7 天 / 30 天涨跌幅） |
| `get_container_detail()` | 武器箱内含物详情 |
| `get_container_roi_detail()` | 武器箱回报率走势 |
| `get_roi_list()` / `get_case_stats()` | 全量武器箱 ROI 排行 |

### 2. 价格走势可视化（K 线图）

**后端（`csqaq_api.py` `get_price_trend()`）：**

将从 CSQAQ 获取的原始 K 线数据按日聚合，生成结构化的价格走势数据：
- 每日 **收盘价、最高价、最低价、在售量**
- 支持 4 个时间周期（30/90/180/365 天）
- 多平台价格对比（Buff / Steam / 悠悠有品）
- 存世量趋势数据

**前端（`index.html` 价格走势 Tab）—— 已集成到主程序：**

基于 **ECharts 专业图表库** 的双面板 K 线图：

| 面板 | 内容 | 说明 |
|:---|:---|:---|
| **上方面板** | 🕯️ K 线蜡烛图 | 红跌绿涨，每根蜡烛显示开盘/收盘/最高/最低 |
| **下方面板** | 📊 成交量柱状图 | 每日在售量，与 K 线时间轴联动 |

**交互功能：**

- **Tooltip 悬停** — 鼠标悬停时显示完整数据（日期、开盘、收盘、最高、最低、成交量）
- **滑块缩放 (dataZoom)** — 底部滑块支持拖动选择时间范围，鼠标滚轮也可缩放
- **时间周期选择** — 支持 30天 / 90天 / 180天 / 365天 四档切换
- **自适应布局** — 切换 Tab 和窗口缩放时 ECharts 自动调整大小
- **数据表格** — 下方同步展示最近 30 天的详细数据（日期、收盘价、最高、最低、在售量）
- **多平台对比** — Buff / Steam / 悠悠有品价格横向对比
- **搜索联动** — 支持中文名称搜索自动补全，点击即查

### 3. 大盘指数看板

**后端（`csqaq_api.py` `get_market_index()`）：**

从 CSQAQ Kline API 获取全饰品市场指数数据：
- 当前指数值
- 今日涨跌幅
- 近 7 天涨跌幅
- 近 30 天涨跌幅
- 5 分钟缓存 + 异常降级

**前端（`index.html` 顶部 `marketBar`）：**

页面顶部实时显示大盘指数条：
```
📊 饰品指数  当前 ¥1740.09  今日 +0.17%  近7天 -5.73%  近30天 -13.26%
```
- 涨为绿色、跌为红色、中性为灰色
- 数据不可用时自动降级显示"大盘指数暂不可用"

### 4. 全量饰品价格爬虫（`crawler.py`）

独立的批量数据采集工具：

| 特性 | 说明 |
|:---|:---|
| 采集方式 | 单线程逐个爬取，间隔 1 秒 |
| 断点续爬 | 进度文件记录已采集 ID，中断后可恢复 |
| 自动保存 | 每 50 件存一次 CSV，防数据丢失 |
| 输出 | 结构化 CSV（id、name、wear、rarity、price、steam_price、rate_7/30/90 等） |
| 容错 | 429 限流自动重试 3 次 |

## 关键技术亮点

1. **K 线数据聚合** — 将原始 tick 级 K 线数据按日聚合，提取每日最高/最低/收盘价，兼顾完整性和性能
2. **数据驱动磨损系数** — 用真实市场价格计算磨损折价系数，替代固定经验值，提高估值模型准确性
3. **断点续爬** — 爬虫支持中断恢复，适合大规模数据采集场景
5. **优雅降级** — 大盘指数不可用时前端自动显示"暂不可用"，不影响其他功能
6. **全链路打通** — 从数据采集（爬虫）→ 数据处理（磨损系数）→ 图表展示，形成完整的数据管道
